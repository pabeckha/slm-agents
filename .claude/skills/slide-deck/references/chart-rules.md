# Chart Rules

## Choosing the Right Chart

| Goal | Chart type |
|------|-----------|
| Compare categories | Horizontal bar (sorted descending) |
| Show change over time | Line chart |
| Show composition | Stacked bar or waterfall |
| Show distribution | Histogram or box plot |
| Show correlation | Scatter plot |
| Show a single number that matters | Big number / scorecard |
| Compare two variables across groups | Dot plot or 2×2 matrix |

Never use: pie charts (use bar), 3-D charts (always), dual-axis line charts (use two slides).

## Chart Anatomy (every chart must have all of these)

1. **Chart title** — states the finding, not the variable ("Revenue grew 23% in Q3", not "Revenue over time")
2. **Axis labels** — include units; no "Value" or "Count" without context
3. **Source line** — bottom-left, small, "Source: [dataset/year]"
4. **Highlight** — one bar, point, or line visually emphasised (color, bold, callout arrow)
5. **Takeaway callout** — one sentence, bottom of chart or callout box

## Color Rules

- One accent color for the key data point; grey for everything else
- Never use red/green together (colorblind)
- Max 3 colors in any single chart

## Axis Rules

- Start quantitative axes at 0 (bar charts); exception allowed for line charts showing trends
- Label every tick that matters; omit labels that clutter
- No gridlines on bar charts; light gridlines only on line/scatter

## Table Rules

- Use a table when exact values matter more than trends
- Bold the row or column with the key finding
- Right-align numbers; left-align text
- One unit per column — never mix % and absolute in the same column

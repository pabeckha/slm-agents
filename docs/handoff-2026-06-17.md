## 1. gemma-3-1b parallel re-run: root-caused and fixed (commit `ed8cdfa`)

The gemma-3-1b `parallel` / `parallel_multiple` cells had failed **twice** — both
infra, never a capability 0%:

1. `GUIDED_BACKEND=outlines` (job 28682151) — died at argument-parse: vLLM 0.8.5
   V1 only offers `{auto, guidance, xgrammar}` (`config.py` `GuidedDecodingBackendV1`).
2. `GUIDED_BACKEND=xgrammar` (jobs 28686566/28686567) — **every case 400'd**:
   "schema contains features not supported by xgrammar". vLLM's xgrammar gate
   (`guided_decoding/utils.py:18-22`) rejects array `minItems`/`maxItems`; with the
   backend *forced* there is no `auto` fallback, so it hard-rejects. The off-HPC
   smoke test gave false confidence because it compiled the **standalone** xgrammar
   lib (which supports those keys), never vLLM's request-path gate.

**The triangle (all three backends fail the parallel schema on gemma):**
- `xgrammar` (forced) → 400, array bounds unsupported.
- `guidance` (llguidance) → gemma 262144-vs-262145 vocab off-by-one crash
  (`backend_guidance.py:84` passes model vocab to `llguidance/hf.py:37`); only
  fixable by patching the installed site-package (rejected: bitmask-width risk +
  NFS-venv revert risk).
- `outlines` → not selectable in V1.

**Fix (Option A — xgrammar + trimmed schema):** drop `minItems`/`maxItems` via
`BFCL_PARALLEL_DROP_ARRAY_BOUNDS=1` (`src/vllm_backend.py`). `oneOf`/`const` are
supported, so the trimmed schema passes — verified directly against the installed
`has_xgrammar_unsupported_json_features` (True→False). The bounds only *weaken* the
constraint (allow 0 or >cap calls), so at temp-0 their removal cannot bias gemma's
score upward.

**Files changed (in `ed8cdfa`):**
- `src/vllm_backend.py` — env-gated bounds drop (default unchanged for all other models).
- `scripts/hpc/run_bfcl_eval.sh` — new results-validity gate: fails the job on a
  client-side 400 (`not supported by xgrammar` / `BadRequestError`) or
  `correct_count=0`. The old gate only watched the *server* log, which is why the
  400 batch slipped through as "done".
- `scripts/hpc/run_gemma_parallel_reruns.sh` — wires the flag; header rewritten with
  the two-failure history.
- `scripts/smoke_parallel_schema.py` — now also checks vLLM's request-path gate.
- `docs/decisions/cross-family-cd-results.md` — two-failure history + backend/schema
  confound to flag in the thesis.

**Quarantined (gitignored, local only):** the four fake-`0%` score files + two

**Confound to flag in the thesis:** other parallel cells ran under `auto` →

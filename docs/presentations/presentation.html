Executive Summary

Three findings, one clear recommendation

DeepGruble is currently paying 20× more than necessary for LLM infrastructure, and a single-vendor dependency threatens the viability of unanswerable.ai as a scalable product. Implementing cascade routing reduces LLM costs by up to 94% and flips medium-client margin from −50% to +71%.
Finding 1 — Commercial Risk

A 50-user client costs $1,200/month in LLM fees alone — more than any viable platform fee. The product cannot scale profitably under the current architecture.

Finding 2 — Strategic Risk

Anthropic controls DG's cost structure, availability, and terms. A 10% price increase or API outage has zero buffer. Competing platforms offering multi-provider support gain a structural advantage.

Finding 3 — Capability Risk

Routing everything through Claude means paying 40× more for simple tasks and missing capabilities like 1M-token context, on-premises privacy, and sub-$0.20/M budget tier providers.

The Solution

Three components deliver full provider independence with a single interface change

Component 1 — Unified Interface

A single TypeScript interface all providers implement. The rest of the codebase never imports a provider SDK directly. Adding a provider = implement one interface, nothing else changes.

Component 2 — Cascade Router

Reads task requirements (privacySensitive, requiresReasoning, contextLength) and routes each request to the optimal-cost provider that meets the quality bar.

Component 3 — Provider Adapters

Six adapters. Three are ~10 lines each because DeepSeek, Grok, and Ollama all speak the OpenAI API format — they extend OpenAIAdapter with just a base URL override.

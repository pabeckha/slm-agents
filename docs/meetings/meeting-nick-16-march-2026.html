Context

The core architecture challenge

⬡
Goal: efficient agentic architecture
Design an architecture where a small language model can reason effectively and call the correct tools.
✗
The problem with small models
SLMs fail at structured output and tool selection out of the box — they hallucinate tool names, produce invalid JSON, and mis-reason about arguments.
✓
Reasoning correctly
The model must decompose tasks, decide when to call a tool vs. respond directly, and chain multi-step reasoning without hallucinating intermediate steps.
✓
Calling the correct tools
Select the right tool from a schema, produce valid arguments, and handle the result — all within constrained compute and latency budgets.
◎
Efficiency constraints
Target: sub-second tool-call latency on consumer hardware. No cloud inference — the model must run locally or on edge devices.
→
Research question
Which combination of constrained decoding, fine-tuning, and inference optimizations closes the gap between SLMs and large frontier models on agentic benchmarks?

Methods

Implementation Order

1
Constrained Decoding
Core thesis claim — PoC done. Run full BFCL eval to establish the baseline.
2
Quantization
No retraining — compress (AWQ INT4). Measure latency and quality delta on top of CD.
3
Inference-Time Compute
No retraining — CoT, ReAct prompting. Cheap to test, strong empirical backing for agentic tasks.
4
RAG
No retraining — retrieve relevant tool schemas at runtime. Reduces hallucination of tool names/args.
5
LoRA / PEFT
Requires training — teach tool-use patterns on ToolBench / Glaive. Highest direct impact.
6
Knowledge Distillation
Requires training — CoT traces from Claude Opus. Builds on the LoRA setup already in place.
7
Pruning
Risky — can degrade instruction-following. Evaluate only after the model is already performing well.
8
Architecture
Benchmarking only — compare pre-trained variants (Flash Attention, MoE). No modifications.
